# /req-story-map

用户故事地图 skill，基于 Jeff Patton 的故事地图方法

## Usage
/req-story-map [产品或功能描述]

## Implementation
创建用户故事地图，提供需求全景视图

### 故事地图结构

```
用户旅程 (User Journey)
    ↓
用户活动 (Activities) ← Backbone
    ↓
用户任务 (Tasks) ← Walking Skeleton
    ↓
用户故事 (Stories) ← 按优先级分层
    ↓
发布规划 (Releases)
```

### 创建步骤

#### 1. 识别用户旅程
从用户视角描述完整的使用流程

**示例**: 在线购物
```
浏览商品 → 选择商品 → 下单 → 支付 → 收货 → 评价
```

#### 2. 分解用户活动 (Backbone)
将旅程分解为主要活动

**示例**: 下单活动
```
- 查看购物车
- 填写收货信息
- 选择支付方式
- 确认订单
```

#### 3. 识别用户任务 (Walking Skeleton)
为每个活动列出具体任务

**示例**: 填写收货信息
```
- 输入收货人姓名
- 输入手机号码
- 选择省市区
- 输入详细地址
- 保存为默认地址
```

#### 4. 编写用户故事
为每个任务编写具体的用户故事

**格式**: 作为 [角色]，我想要 [功能]，以便 [价值]

**示例**:
```
- 作为购买者，我想要保存常用地址，以便下次快速下单
- 作为购买者，我想要验证手机号格式，以便避免输入错误
- 作为购买者，我想要从地址簿选择地址，以便快速填写
```

#### 5. 优先级排序
将故事按优先级垂直排列

```
高优先级 (MVP)
├─ 必须有的核心功能
│
中优先级 (Release 2)
├─ 重要但非必需
│
低优先级 (Future)
└─ 锦上添花的功能
```

### 故事地图模板

```markdown
# 用户故事地图: [产品/功能名称]

## 用户角色
- [角色 1]: [描述]
- [角色 2]: [描述]

## 用户旅程
[步骤1] → [步骤2] → [步骤3] → [步骤4]

---

## Backbone (用户活动)

### [活动 1]
**Walking Skeleton (任务)**:
- [ ] [任务 1.1]
- [ ] [任务 1.2]
- [ ] [任务 1.3]

**用户故事**:

**MVP (Release 1)**:
- [ ] 作为 [角色]，我想要 [功能]，以便 [价值]
- [ ] 作为 [角色]，我想要 [功能]，以便 [价值]

**Release 2**:
- [ ] 作为 [角色]，我想要 [功能]，以便 [价值]

**Future**:
- [ ] 作为 [角色]，我想要 [功能]，以便 [价值]

---

### [活动 2]
[重复上述结构]

---

## 发布规划

### MVP (最小可行产品)
目标: [MVP 的核心价值]
包含故事: [数量]
预计工作量: [估算]

### Release 2
目标: [第二版的增强价值]
包含故事: [数量]
预计工作量: [估算]

### Future
目标: [未来的愿景]
包含故事: [数量]
```

### 可视化表达

使用表格形式表达故事地图：

```
| 活动1      | 活动2      | 活动3      | 活动4      |
|-----------|-----------|-----------|-----------|
| 任务1.1    | 任务2.1    | 任务3.1    | 任务4.1    |
| 任务1.2    | 任务2.2    | 任务3.2    | 任务4.2    |
|-----------|-----------|-----------|-----------|
| 故事1.1 ⭐ | 故事2.1 ⭐ | 故事3.1 ⭐ | 故事4.1 ⭐ | ← MVP
| 故事1.2    | 故事2.2    | 故事3.2    | 故事4.2    | ← Release 2
| 故事1.3    | 故事2.3    | 故事3.3    | 故事4.3    | ← Future
```

### 关键原则

1. **用户视角**: 从用户的角度组织故事
2. **时间顺序**: 按照用户使用的时间顺序排列
3. **完整性**: 覆盖完整的用户旅程
4. **优先级**: 垂直方向表示优先级
5. **协作**: 团队共同创建和维护

### 使用场景

- **产品规划**: 规划产品路线图
- **发布规划**: 确定每个版本包含的功能
- **需求沟通**: 与利益相关者沟通需求全景
- **迭代规划**: 选择下个迭代的故事
- **差距分析**: 识别缺失的功能

### 质量检查

- [ ] 用户旅程完整
- [ ] 活动按时间顺序排列
- [ ] 任务覆盖活动的主要步骤
- [ ] 故事符合 INVEST 原则
- [ ] 优先级清晰
- [ ] MVP 可以独立交付价值

### 输出
生成用户故事地图文档，包括：
- 用户旅程图
- 活动和任务分解
- 用户故事列表
- 优先级标注
- 发布规划
